# Anti-Theft Runbook

How this site defends against being cloned/scraped, and what *you* (the operator) do
to detect and act on copies. The strategy is **attribute + detect**, not "force" —
because cited research (June 2026) showed hidden-instruction injection is mostly
ineffective against the paste-into-an-LLM threat and risks our own SEO. We do not
inject hidden prompts, run a cloaking edge function, or use tarpits.

---

## What's built (in the codebase)

| Layer | Where | What it does |
|---|---|---|
| **Self-canonical** | `src/components/Seo.tsx` (per-route), `index.html` (home) | Every page declares its own canonical URL. A verbatim copy keeps it → Google credits *us* as origin and ranks the copy below. This is the main anti-theft weapon. |
| **Copyright structured data** | `src/lib/attribution.ts` → `withArticleAttribution()` applied in the blog/case-study layouts | Adds `copyrightHolder`, `copyrightNotice`, `creditText`, `license`, `usageInfo`, `author`, `publisher` to each Article's JSON-LD. Machine-readable provenance; mild AI-citation upside; zero SEO risk. |
| **Authorship meta** | `index.html` + `Seo.tsx` | `author`, `copyright`, `twitter:creator` tags. |
| **Visible attribution line** | `src/components/ArticleAttribution.tsx` (foot of every article) | The only watermark that survives readability extraction and screenshots. Carries the per-page **canary ref**. |
| **Canary tracking pixel** | `src/components/CanaryPixel.tsx` (needs setup → below) | Invisible image that emails you the IP/host/UA when a copy is loaded in a browser. |
| **License terms** | `src/pages/LicensePage.tsx` → `/license` | All-rights-reserved + attribution-required terms; referenced by JSON-LD `license`/`usageInfo`. |
| **Build guard** | `scripts/prerender.mjs` | Warns at build time if any prerendered page loses its canonical or copyright JSON-LD. |

## One-time setup: activate the canary pixel

The pixel is wired up but **disabled until you paste a token URL**.

1. Go to <https://canarytokens.org>.
2. Select token type **"Web bug / URL token"**.
3. Enter your email (where alerts go) + a memo, e.g. `naqeebali.me page-load canary`.
4. Copy the generated **image URL** (looks like `https://canarytokens.com/.../contact.png`).
5. Paste it into `src/lib/attribution.ts`:
   ```ts
   export const CANARY_PIXEL_URL = 'https://canarytokens.com/.../contact.png';
   ```
6. Rebuild + deploy. From then on, every page embeds the pixel 1px off-screen.

**What fires it:** someone loads a verbatim copy of one of your pages (including the
pixel `<img>`) in a real browser/agent. You get an email with their IP, host, and
user-agent. **What won't fire it:** a pure copy-the-text-into-ChatGPT workflow, or a
cloner who strips remote images. It's a high-confidence signal when it triggers, not
total coverage.

> Note: a single token answers "is a copy of *some* page of mine being served?" Per-page
> granularity comes with the deferred self-hosted endpoint (Phase 2, below).

## Canary ref tokens (trap-street strings)

Every article footer shows a deterministic token like `ref nqb-1a2b3c` (generated by
`canaryRef(slug)` in `src/lib/attribution.ts`). It's stable across builds, so any
appearance of that exact token **off** `naqeebali.me` is a verbatim copy — and the
token tells you *which page* was copied.

- **Forensic use (reliable):** when you suspect a copy, Ctrl-F the page for the ref
  token. If present, it's a verbatim clone of that specific page — solid DMCA evidence.
- **Proactive use (best-effort):** periodically search Google for a token, e.g.
  `"nqb-1a2b3c" -site:naqeebali.me`. Hyphenated-token matching is imperfect, so treat
  this as a bonus, not your primary alarm (use the alerts + Copyscape below for that).

## Detection routine

### Google Alerts (set once, free)
Create alerts at <https://www.google.com/alerts>:
1. `"Naqeebali Shamsi"` — name mentions.
2. One distinctive **full sentence** from each high-value page (a sentence unlikely to
   occur by chance). These catch indexed verbatim/near-verbatim copies far more
   reliably than token search. Example pattern: a unique opening line of a case study.
3. Optionally a couple of ref tokens for your top pages.

### Copyscape (monthly, free tier)
Run your top URLs through <https://www.copyscape.com> once a month. It's the industry
standard for finding indexed copies. For LLM-*paraphrased* copies (which Copyscape can
miss), Copyleaks/Originality.ai do similarity detection but are paid — only escalate if
paraphrased theft becomes a real problem.

### Server/CDN logs (optional)
Netlify exposes user-agent categories. Watching for declared AI crawlers tells you
you're being *crawled* (expected — we welcome citation), not that you've been *copied*.

## When you find a copy

1. **Confirm** it's verbatim: find the ref token and/or trigger the pixel.
2. **Snapshot evidence:** save the offending URL, a screenshot, and the matching ref
   token / pixel hit (date, IP).
3. **DMCA takedown** — send to the host and to Google
   (<https://support.google.com/legal/troubleshooter/1114905> → Web Search).

### DMCA template
```
Subject: DMCA Takedown Notice

I am the copyright owner of content located at [ORIGINAL URL on naqeebali.me].
The following URL reproduces my copyrighted work without authorization:
  Infringing URL: [COPY URL]
  My original:    [ORIGINAL URL]
  Proof of origin: my page is the canonical source (see <link rel="canonical">),
  carries my authorship/copyright structured data, and contains the unique
  identifier "[REF TOKEN]" which the copy reproduces verbatim.

I have a good-faith belief that the use is not authorized by the copyright owner,
its agent, or the law. The information in this notice is accurate, and under penalty
of perjury, I am the copyright owner or authorized to act on the owner's behalf.

Name: Naqeebali Shamsi
Contact: [email]
Signature: /Naqeebali Shamsi/
Date: [date]
```

## Phase 2 (deferred): self-hosted canary endpoint

Replaces the third-party pixel with one you control, for per-page granularity +
referrer capture and no external dependency.

- Add `netlify/functions/canary.js` (or an edge function) that logs
  `referer`, `x-forwarded-for`, `user-agent`, timestamp, and a `?p=<slug>` param to a
  store you control (Netlify Blobs / a logging webhook), then returns a 1×1 GIF.
- Point `CANARY_PIXEL_URL` at `/.netlify/functions/canary?p=` + per-page slug (make
  `CanaryPixel` take a `slug` prop and append it).
- Netlify free tier easily covers a portfolio's volume (functions billed in the shared
  credit pool; no overage billing — it pauses rather than charges).

## Why we did NOT build the "force a backlink" version

Cited findings from the planning session:
- A hidden injected instruction pasted into ChatGPT + Perplexity had **zero effect**
  (Search Engine Land, Sep 2025).
- Mozilla Readability strips `display:none` / `aria-hidden` / `<script>` before the
  model sees them.
- LLMs now rank the user's "rebuild this" prompt above page text; a "link to me"
  instruction is the easiest to drop.
- Poison/watermark schemes lose to sanitizers (Nightshade → defeated by LightShed).

Net: hidden-instruction injection is theater for this threat and would have risked our
own SEO. Provenance + detection is the version that actually protects the work.
