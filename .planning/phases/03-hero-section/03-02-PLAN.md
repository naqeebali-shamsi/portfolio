  <name>Task 1: Install R3F ecosystem and create GlassIcosahedron + HeroScene components</name>
  <files>
    package.json
    src/components/Hero/GlassIcosahedron.tsx
    src/components/Hero/HeroScene.tsx
  </files>
  <action>
    1. Install Three.js and R3F: `npm install three @react-three/fiber @react-three/drei`
       - Also install types: `npm install -D @types/three`
       - Verify R3F v9 is installed (required for React 19 compatibility). If npm resolves v8, use `npm install @react-three/fiber@^9`.

    2. Create `src/components/Hero/GlassIcosahedron.tsx`:
       - Import `Icosahedron` and `MeshTransmissionMaterial` from `@react-three/drei`.
       - Import `useFrame` from `@react-three/fiber`.
       - Accept a `mouse` prop: `React.RefObject<{ x: number; y: number }>` — normalized mouse position (-1 to 1).
       - Icosahedron geometry: use `<Icosahedron args={[1.5, 0]}>` (detail=0 for clean facets). Adjust size as needed for visual balance.
       - Material: `<MeshTransmissionMaterial>` with props:
         - `color="#70ABAF"` (accent color for tint)
         - `transmission={0.95}` (high transparency)
         - `roughness={0.1}`
         - `thickness={0.5}`
         - `chromaticAberration={0.03}`
         - `resolution={256}` (keep low per research — backside doubles cost)
         - `samples={6}`
         - `backside={false}` (avoid double render cost)
       - `useFrame` logic:
         - Gentle cursor follow: lerp mesh rotation toward mouse position. Use `THREE.MathUtils.lerp` with a small factor (~0.02-0.05) for smooth, non-aggressive follow.
         - Target rotation: `mouse.current.x * 0.3` for Y axis, `mouse.current.y * 0.3` for X axis.
         - Ambient rotation: Always add a small constant rotation (~0.001-0.003 per frame) to Y axis so it's never static even without cursor movement.
       - Use `useRef` for the mesh reference.

    3. Create `src/components/Hero/HeroScene.tsx`:
       - This is the component that will be lazy-loaded. It contains the R3F `<Canvas>`.
       - Import `Canvas` from `@react-three/fiber`.
       - Import `GlassIcosahedron`.
       - Track mouse position: Use `onPointerMove` on a wrapper div to update a ref `{ x, y }` with normalized coordinates (-1 to 1, based on wrapper dimensions).
       - Canvas setup:
         - `<Canvas camera={{ position: [0, 0, 5], fov: 45 }}>`
         - `gl={{ alpha: true, antialias: true }}` for transparent background.
         - `style={{ background: 'transparent' }}`.
       - Inside Canvas: `<GlassIcosahedron mouse={mouseRef} />`.
       - Add basic lighting: `<ambientLight intensity={0.5} />` and `<directionalLight position={[5, 5, 5]} intensity={1} />`.
       - Do NOT add `<Environment>` initially (per research: "Start without, add if glass looks flat"). If the glass looks too flat during testing, add `<Environment preset="studio" />` from drei.
       - IntersectionObserver for WebGL disposal (HERO-05):
         - Use a `useRef` for the wrapper div.
         - In useEffect, create an IntersectionObserver on the wrapper div.
         - Track `isVisible` state. Use `rootMargin: "100px"` to add buffer (prevents mount/unmount flicker per research).
         - When `isVisible` is false, unmount the Canvas entirely (conditional rendering). Unmounting Canvas disposes the WebGL context — do NOT use `frameloop="never"` (per research: it does NOT dispose WebGL).
         - When `isVisible` becomes true again, Canvas remounts.
       - Export as default (for React.lazy compatibility).
  </action>

  <name>Task 2: Wire lazy-loaded 3D into Hero.tsx with device detection gating</name>
  <files>
    src/components/Hero/Hero.tsx
  </files>
  <action>
    1. Update `src/components/Hero/Hero.tsx`:
       - Add lazy import at top of file (OUTSIDE the component):
         ```
         const HeroScene = React.lazy(() => import('./HeroScene'));
         ```
         This ensures three.js (~600KB) stays out of the main bundle. The Canvas MUST be inside the lazy-loaded component (per research: "Canvas in parent, scene in lazy child = three.js in main bundle").

       - Read `canRender3D` from `useDeviceCapability()` (already imported from Plan 01).

       - Update the right-side column rendering logic:
         ```
         {canRender3D ? (
           <Suspense fallback={<HeroCSSFallback />}>
             <HeroScene />
           </Suspense>
         ) : (
           <HeroCSSFallback />
         )}
         ```

       - On low-end devices (`canRender3D === false`): HeroCSSFallback renders permanently. React.lazy never triggers the import, so three.js never downloads.
       - On capable devices: HeroCSSFallback shows instantly as Suspense fallback while HeroScene chunk loads. Once loaded, HeroScene replaces it (instant swap, no crossfade).

    2. Verify the lazy boundary:
       - Run `npm run build` and check the output. There should be a separate chunk for HeroScene (look for a chunk containing three.js). The main bundle should NOT contain three.js.
       - In the build output, look for chunk size info. The HeroScene chunk will be large (~600KB+ uncompressed) and that's expected — it's lazy-loaded.
  </action>

    - Scrolling past the hero and back: Canvas unmounts and remounts (check browser devtools Performance tab or console log if added).
