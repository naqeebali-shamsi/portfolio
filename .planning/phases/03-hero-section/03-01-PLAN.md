    3. Create `src/hooks/useDeviceCapability.ts`:
       - Export a hook `useDeviceCapability()` that returns `{ canRender3D: boolean }`.
       - Check `navigator.hardwareConcurrency >= 4` (92% browser support).
       - Check `navigator.deviceMemory >= 4` (Chromium-only). Default to `8` if `navigator.deviceMemory` is undefined (Safari/Firefox don't expose it — default to capable per research).
       - If BOTH checks pass, `canRender3D = true`. Otherwise `false`.
       - Compute once on mount (useMemo or useState with empty deps). No re-computation needed.
       - Type the navigator extensions properly (deviceMemory is not in standard types — use interface merging or type assertion).
  </action>

    - `npx tsc --noEmit` passes (no type errors in content.ts or useDeviceCapability.ts)
  </verify>

    GSAP is installed, hero content exists in content.ts, and useDeviceCapability hook returns { canRender3D: boolean } based on hardware detection.
  </done>

       - Implementation approach: Maintain a `displayText` state. Use gsap.to with an object `{ index: 0 }` to animate the character index, updating displayText via onUpdate. Type forward, pause (~1.2s), delete backward, pause (~0.3s), move to next title.

       - Loop infinitely: `timeline.repeat(-1)` or rebuild timeline on completion.
       - Render a `<span>` with the current text plus a blinking cursor character (pipe `|` or thin block, animated with CSS).
       - Use `gsap.context()` for cleanup. Return `ctx.revert()` in useEffect cleanup.

    3. Create `src/components/Hero/HeroCSSFallback.tsx`:
