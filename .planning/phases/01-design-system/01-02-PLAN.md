  <action>
    1. Create `src/hooks/useScrollDirection.ts`:
       - TypeScript hook returning `{ direction: 'up' | 'down' | null, isAtTop: boolean }`
       - Uses requestAnimationFrame for debouncing (NOT raw scroll events)
       - 10px threshold to prevent jitter from tiny scroll amounts
       - Passive scroll listener for performance
       - Cleans up listener on unmount
       - Follow the pattern from 01-RESEARCH.md exactly

    2. Create directory `src/components/Navbar/` if it doesn't exist.

    3. Create `src/components/Navbar/Navbar.tsx`:
       - Import and use `useScrollDirection` hook via `@/hooks/useScrollDirection`
       - Import `clsx` for conditional classes (already in package.json)

       **Desktop layout (md and above):**
       - Fixed position, full width, z-50
       - Height: h-16 (4rem)
       - Left side: "NS" in font-heading, text-lg, font-bold, tracking-heading
       - Right side: 4 links — "How I Build" (#how-i-build), "Work" (#work), "Experience" (#experience), "Contact" (#contact)
       - Links styled: font-body, text-sm, text-text-muted, hover:text-accent, transition-colors
       - Max width constrained to max-w-container with mx-auto and px-6

       **Scroll behavior:**
       - When `isAtTop === true`: nav is transparent (no background, no border)
       - When scrolled (isAtTop false) AND direction is 'up' or null: show nav with `bg-bg/80 backdrop-blur-md border-b border-accent-light/20`
       - When direction is 'down' AND not at top AND mobile menu not open: hide with `-translate-y-full`
       - Transition: `transition-transform duration-300 ease-smooth`

       **Mobile layout (below md):**
       - Hamburger button replaces links: 3 horizontal lines (spans), w-6 h-5
       - On click, toggle `mobileOpen` state
       - Hamburger animates to X when open: top line rotates 45deg + translates, middle line fades, bottom line rotates -45deg + translates
       - Full-screen overlay: `fixed inset-0 z-40 bg-bg`, centered flex column
       - Overlay links: font-heading, text-4xl, font-bold, uppercase, tracking-heading, hover:text-accent
       - Clicking any overlay link closes the menu (setMobileOpen(false))
       - When mobile menu is open, nav should NOT hide (isHidden check includes `!mobileOpen`)

       **Accessibility:**
       - Hamburger button: `aria-label="Toggle menu"`
       - Nav element uses `<nav>` semantic tag
       - Links are standard `<a>` tags with href anchors

    4. Update `src/App.tsx`:
       - Import Navbar from `@/components/Navbar/Navbar`
       - Add `<Navbar />` at the top of the component, before `<main>`
       - Add `id` attributes to each placeholder section matching the nav anchors:
         - First section (hero/name): no id needed (top of page)
         - "HOW I BUILD" section: `id="how-i-build"`
         - "WORK" section: `id="work"`
         - "EXPERIENCE" section: `id="experience"`
         - "CONTACT" section: `id="contact"`
       - Add `pt-16` to the first section (or the outer div) to account for the fixed nav height, so content isn't hidden behind the nav

    IMPORTANT: Use `@/hooks/useScrollDirection` and `@/components/Navbar/Navbar` import paths — the `@` alias is configured in both vite.config.js and tsconfig.json.

    IMPORTANT: The nav's z-index must be z-50. Future 3D canvas in the hero will be z-0 or z-10. This prevents z-index conflicts noted in research pitfalls.
