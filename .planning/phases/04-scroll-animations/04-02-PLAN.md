  <name>Task 1: Parallax on Hero + Case Study pin-and-scrub</name>
  <files>src/components/Hero/Hero.tsx, src/sections/CaseStudy.tsx</files>
  <action>
**Hero parallax:**
- In Hero.tsx, add a useGSAP call (import from '@/lib/gsap') with scope on the section ref
- Wrap in gsap.matchMedia() — only apply when prefers-reduced-motion: no-preference
- Apply subtle parallax to the text container: `yPercent: 5` with `scrub: true`, trigger is the hero section, start 'top top', end 'bottom top'
- Apply opposite parallax to the 3D/fallback container: `yPercent: -10` with `scrub: true` (same trigger)
- This creates a depth sandwich — foreground text drifts down slightly, background 3D element drifts up
- Intensity is 5-15% as specified in CONTEXT.md
- Use `immediateRender: false` on scrubbed animations to avoid Pitfall 5

**Case Study pin-and-scrub (DESKTOP ONLY):**
- Replace or enhance the existing CaseStudy reveal animations from Plan 01 with a pin-and-scrub approach on desktop
- Use gsap.matchMedia() with conditions: `desktop: '(min-width: 768px) and (prefers-reduced-motion: no-preference)'` and `mobile: '(max-width: 767px) and (prefers-reduced-motion: no-preference)'`
- Desktop behavior:
  - Pin the outermost section element (NOT a nested child — avoids pin-spacer layout breaks)
  - Create a GSAP timeline with `scrub: 1` (smooth 1s catch-up), `start: 'top top'`, `end: '+=200%'`
  - Timeline sequence: (1) intro fades in, (2) intro slides up/fades as architecture block reveals, (3) architecture slides up/fades as approach block reveals
  - Use gsap.set() for initial hidden states on approach and architecture blocks
  - Use `.fromTo()` instead of `.from()` for scrubbed animations (avoids immediateRender conflict)
- Mobile behavior:
  - Keep the simple stagger reveals from Plan 01 (no pin, no scrub)
  - Each block (intro, architecture, approach) fades up independently on scroll
- Add CSS classes to the three CaseStudy content blocks for targeting: `cs-intro`, `cs-architecture`, `cs-approach`

Important: The pin must be on the section element itself. Test mentally that the section contains all three blocks and the full-width breakout div.
  </action>

3. On dev server at desktop width: case study should pin in place and sequentially reveal content as you scroll through
4. Resize to mobile width: case study should NOT pin, each block reveals independently
5. Enable prefers-reduced-motion — no parallax, no pin, content visible immediately
