<task type="auto">
  <name>Task 2: Add ScrollTrigger reveal animations to all content sections</name>
  <files>src/sections/HowIBuild.tsx, src/sections/CaseStudy.tsx, src/sections/Experience.tsx, src/sections/Contact.tsx</files>
  <action>
Add scroll-reveal animations to each section using the useGSAP hook with scope. All animations follow these rules:
- Wrapped in `gsap.matchMedia()` — skip all animations when `(prefers-reduced-motion: reduce)` is active
- Use `once: true` on all ScrollTriggers — no replay on scroll back
- Use `gsap.set()` for initial invisible state to prevent FOUC (flash of unstyled content)
- Easing: `cubic-bezier(0.16, 1, 0.3, 1)` (the --ease-reveal curve)
- Duration: 600-800ms for scroll reveals
- Import { useGSAP } from '@gsap/react' and gsap from '@/lib/gsap'

**HowIBuild.tsx:**
- Add a useRef for the section element, pass as scope to useGSAP
- gsap.set() all statement paragraphs to opacity: 0, y: 60 on mount
- gsap.to() with stagger: 0.15 on each statement — fade up (opacity: 1, y: 0)
- ScrollTrigger start: 'top 80%', once: true
- Add a CSS class like `statement` to each paragraph for targeting (or use a data attribute)

**CaseStudy.tsx:**
- Add useRef for the section
- Three separate reveals: (1) intro block (title + problem), (2) full-width architecture/screenshots block, (3) approach block
- Each reveals with fade+slide (opacity 0->1, y 40->0), triggered independently as each block enters viewport
- Intro: start 'top 80%'. Architecture block: start 'top 75%'. Approach: start 'top 85%'.

**Experience.tsx:**
- Add useRef for the section
- Stagger-reveal timeline entries — each entry fades up with stagger: 0.12
- The heading reveals first, then entries stagger in
- Education block: separate trigger, simple group fade

**Contact.tsx:**
- Simple group reveal — all content fades up together (no stagger needed, it's a compact section)
- Start: 'top 80%'

For all sections: add `useRef<HTMLElement>(null)` and attach to the root `<section>` element. Use `{ scope: sectionRef }` in useGSAP.
  </action>

  </verify>
