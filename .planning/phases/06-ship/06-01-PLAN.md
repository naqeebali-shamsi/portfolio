Apply mobile-first responsive adjustments to each section. Use Tailwind breakpoints: unprefixed = mobile, sm: = 640px, lg: = 1024px, xl: = 1280px. Study blunar.cz's mobile spacing philosophy: generous vertical rhythm, tighter horizontal padding on small screens, typography that scales down gracefully.

**HowIBuild.tsx:**
- Reduce container padding on mobile: `px-5 sm:px-6 lg:px-8`
- Tighten vertical spacing between statements on mobile: `space-y-10 sm:space-y-16 lg:space-y-24` (currently space-y-16 md:space-y-24)
- Font sizes are already responsive (text-3xl md:text-4xl lg:text-5xl) -- verify they look right at 375px

**CaseStudy.tsx:**
- Container padding: `px-5 sm:px-6 lg:px-8` (same pattern throughout)
- Architecture section grid: already `lg:grid-cols-2` so stacks on mobile -- good
- Device frames: Show only ONE device frame on mobile, both on sm+. Wrap the flex container: hide the second DeviceFrame below sm: with `hidden sm:block` on the second one. On mobile show single centered frame.
- Architecture flow diagram: The ArchBox width `w-52` may be too wide on small screens. Make it `w-40 sm:w-52`. The two-branch flex with `gap-8` may overflow -- use `gap-4 sm:gap-8`.
- Heading sizes: `text-3xl sm:text-4xl lg:text-5xl` (currently text-4xl md:text-5xl)

**Experience.tsx:**
- On touch devices, expand entries on TAP (not just hover). The current `onPointerEnter`/`onPointerLeave` with `supportsHover` guard means touch users can never see details. Add an `onClick` handler that toggles expansion on touch devices: if `!supportsHover.current`, clicking toggles the entry open/closed. Track which entry is open with state.
- Container padding: `px-5 sm:px-6 lg:px-8`

**Contact.tsx:**
- Email button text: already responsive (text-2xl md:text-4xl lg:text-5xl) -- good
- Container: `min-h-[80vh] sm:min-h-screen` -- don't force full viewport height on small mobile where content is compact
- Social links wrapper already centered -- good

**SocialLinks.tsx:**
- Add minimum touch target: each link gets `min-h-11 min-w-11 flex items-center justify-center` to ensure 44x44px tap area
- Increase gap slightly on mobile for fat-finger safety: `gap-8 sm:gap-6` (yes, wider on mobile)

**HeroText.tsx:**
- The display font uses `clamp(4rem, 8vw, 8rem)` which is already responsive. At 375px this resolves to ~4rem (64px) which is good.
- Add bottom spacing for mobile hero: the text block should have adequate breathing room since no 3D visual sits beside it on mobile. Add `lg:pb-0 pb-8` or similar if needed after visual check.

**Navbar.tsx:**
- Mobile overlay links: ensure tap targets are adequate. The `text-4xl` links with `gap-8` spacing should be fine. Add `min-h-11` to be safe.
- Hamburger button: ensure it has adequate tap target. Currently `w-6 h-5` -- wrap in padding or use `p-2` to expand tap area to 44px: change to `p-3 -m-3` (visual size stays same, tap area expands).

Do NOT change desktop layouts. Only add mobile-first base styles and sm:/lg: overrides where the current defaults are desktop-optimized.
